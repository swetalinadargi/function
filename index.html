Que 4 :Print the Output.

   var x = 21;
    var girl = function () {
        console.log(x);
        var x = 20;
    };
    girl ();

Ans:undefined

        We know that if we declare variable outside of the function(means Global 
        Variable) then it'll be accessible & updatable in the whole program anywhere.
        
        And we know that we can declare variable again and again by using var 
        keyword. But with the let keyword we can't declare more than once.
        
        Right declaration with ""var"" keyword: var x= 10; var x=11;   // Value will be of x
        11 with second time declared variable and first time declared variable will not be
        accessable or it has been removed already when we declare the second time.  
        
        Wrong declaration with ""let"" keyword: let n=10; let n=11; // we can't declare 
        variable again by using let keyword. it'll give error.
        
        But if we declare variable with same name in the function as we declare in
        the Global variable then variable which is present inside the function it'll work 
        as local variable(Function Scope). 
        
        That's why here x will print the "undefined".

Que 5 :Print the Output.

    var x = 21;
    girl ();
    console.log(x)
    function girl() {
        console.log(x);
        var x = 20;
    };

Ans:undefined & 21

    We knows the hoisting rule in JS if any variable we want to access before it's 
    declaration & we declare variable after the accessible statement it is possible because
    Hoisting handles this. It brings the declaration above.
    
    Note: it's only bring the declaration above not bring the intialization & assignment above.
    Ex:- 
    console.log(x);  //accessible statement & we are accessing variable before its declaration
    var x =20;  // here it'll bring only declaration means var = x ; 
    
    That's why here our answer will be :
    undefined
    21
